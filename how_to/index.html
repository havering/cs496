Ionic How-To

Introduction

This how-to will walk the user through creating an Ionic app from scratch, using OSX as an operating system and the iOS emulator built into Xcode to emulate the application on an iPhone. The instructions here assume a basic familiarity with dynamically-typed programming languages, a previously-installed text editor as well as Xcode, using the Terminal, and the Chrome browser.

Ionic has two releases with significantly differing syntax as Ionic 1 is built on Angular 1, and Ionic 2 is built on Angular 2. For this how-to, we'll be using Ionic 1 and thus Angular 1 syntax.

Getting Started

To start off, you'll need to install some important components. Head over to NodeJS.org and install Node.js, which will come with Node's packet manager, npm. The easiest way to install Node is through the installer, so it is suggested that you use it here. Once installed, you can check that the installation succeeded by typing

npm -v

in the terminal. It should return a version number. If the command fails, attempt the installation again.

Following successful installation of npm, you are ready to install Ionic. You have the option of installing Ionic either globally, on your entire system, or locally, in the project folder. In general, it is suggested that you install a package globally if you're going to be using the command line to interact with it. This will be the case with Ionic, so it is suggested that you install it globally with the command

npm install -g ionic

We're going to be building an Ionic app that uses Cordova's geolocation, as well as several other Cordova enhancements, so you will also want to install Cordova globally with a similar command:

npm install -g cordova

Now it's time to generate your Ionic app! In the terminal, navigate to the folder in which you'd like your Ionic app to be created. Ionic has a variety of pre-made templates that you can choose from, ranging from very basic to doing almost all of the styling for you. The more complex templates are pretty overkill for this example, and take away some of the fun of doing the styling yourself, so we're going to use the very basic side menu template and expand from there. To initialize an Ionic app with the side menu template, use the command

ionic start how_to sidemenu

Ionic will start downloading required packages via npm, so make sure that you're connected to the internet. Some of the packages downloaded will have warnings about them being deprecated - while that might be true, and it may be possible to individually update the packages, your app will function fine with the deprecated packages so feel free to ignore the warnings.

Once everything has been downloaded, you should find that a folder has been created with the name how_to. Navigate inside the folder, then type

ionic serve

After a momentary pause, Ionic will start the webserver that serves a live version of your app, and launch your app in Chrome. If you accidentally close the browser window, you can navigate back to it by going to localhost with the default Ionic port:

http://localhost:8100/

The app will redirect you to the default index, which, in the case of the sidemenu template, is

http://localhost:8100/#/app/playlists

And now you're done, right? Only if you want a monochromatic app that serves up some sort of playlists. We don't, so it's time to move onto customizating and expanding our app.
